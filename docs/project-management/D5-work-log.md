## 当日目标

建立数据源审计与数据处理方案，明确思明区行政边界、鼓浪屿处理、底图选择、天地图凭证管理、OSM 路网来源、双路网筛选、坐标系统和数据质量审计规则。

## 已确认决策

| 决策项 | 结果 |
|---|---|
| 研究范围 | 福建省厦门市思明区，包含鼓浪屿 |
| 边界策略 | 主边界源 + 辅助边界源空间对照 |
| 鼓浪屿 | 纳入研究范围，保持真实独立路网，不建立虚假跨海连接 |
| 默认底图 | 天地图 |
| 备用底图 | OpenStreetMap |
| 离线兜底 | 简化本地底图或空白底图 |
| 底图凭证 | 通过环境变量或本地配置管理，不提交 GitHub |
| 分析数据源 | OpenStreetMap 路网数据 |
| 首版获取方式 | OSMnx，记录 Overpass 与 PBF 离线方案作为演进路线 |
| 数据快照 | 使用固定获取时间的数据版本 |
| 双路网 | 从 OSM 原始道路派生机动车路网与步行路网 |
| 坐标策略 | 存储与 API 使用 EPSG:4326，米制计算使用 EPSG:32650 |
| 审计等级 | Blocker、Warning、Info |

## 审计重点

- 不把底图凭证提交到公开仓库；
- 不把底图当作分析数据源；
- 不建立鼓浪屿到厦门岛主体的虚假道路连接；
- 不直接使用 EPSG:4326 进行米制长度、面积和密度计算；
- 不将 OSM 最新实时数据作为不可追踪输入；
- 不将 D5 扩展为 POI、实时交通或设施可达性分析；
- 所有正式数据必须绑定数据版本。

## 遗留事项

- D6 需要搭建工程骨架和基础配置文件；
- D6/D7 需要实现 `.env.example`，但不得包含真实底图凭证；
- D7 或后续 Pipeline 阶段需要实际获取思明区边界和 OSM 路网，并输出 D5 规定的审计统计；
- D5 确认的 `EPSG:32650` 在正式计算前仍需通过实际数据范围和投影检查复核。

## D5 结论

D5 已形成数据源审计与数据处理方案。该方案与 D1-D4 保持一致，为 D6 工程骨架搭建、D7 Pipeline 开发、PostGIS 入库和后续指标计算提供了数据治理基线。
