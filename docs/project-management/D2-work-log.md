# D2 工作记录：需求分析

日期：2026-06-23
分支：`docs/d2-requirements-analysis`

## 当日目标

将 D1 项目章程转化为可编号、可验收、可追踪的需求规格，为 D3 总体设计建立稳定输入。

## 已确认决策

| 决策项 | 结果 |
|---|---|
| 用户权限 | 匿名只读，不建设登录与后台管理 |
| 综合权重 | 固定、公开，可查看单项指标 |
| 结果导出 | 分别导出当前筛选后的道路或网格 CSV |
| 底图方案 | 天地图默认，OSM 备用，本地兜底 |
| 道路检索 | 名称模糊搜索、定位和高亮 |
| 数据更新 | 固定版本分析，人工脚本更新 |
| 网格解释 | 原始值、标准化值、综合分、百分位、全区均值比较 |
| 异常区域 | 高值与低值榜单，点击定位 |
| 统计图表 | 指标卡、分布柱状图、网格雷达图 |
| 中心性指标 | 边介数中心性 |
| 路网模型 | 机动车路网与步行路网分离 |
| 离线能力 | 业务功能离线可用，使用简化或空白底图兜底 |
| 导出范围 | 当前图层经过筛选后的全部结果 |
| 文档方法 | 需求编号 + 核心用例 + 追踪矩阵 |
