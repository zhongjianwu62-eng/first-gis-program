# D5 数据处理方案

## 1. 处理目标

D5 数据处理方案用于指导后续 Pipeline 开发。首版目标是从可复现边界和 OSM 路网数据生成 PostGIS 可用的数据版本，包括道路、节点、500 米网格、双路网标记和审计统计。

## 2. 输入数据

| 输入 | 来源 | 用途 |
|---|---|---|
| 思明区行政边界 | 主边界源 + 辅助对照源 | 裁剪道路、生成网格、标记边界网格 |
| OSM 道路数据 | OSMnx / Overpass | 路网结构分析和可步行性指标输入 |
| 底图配置 | 天地图 / OSM / 离线兜底 | 前端展示背景，不参与指标计算 |

## 3. 处理流程

```text
准备行政边界
        ↓
获取 OSM 道路数据
        ↓
统一 CRS 与几何合法性检查
        ↓
投影到 EPSG:32650
        ↓
裁剪研究范围
        ↓
道路类型归一化
        ↓
访问属性筛选
        ↓
生成机动车路网与步行路网标记
        ↓
构建节点和网络图
        ↓
生成 500 米网格
        ↓
统计数据质量指标
        ↓
几何转回 EPSG:4326
        ↓
写入 PostGIS 候选数据版本
        ↓
审计通过后发布当前版本
```

## 4. 边界处理

边界处理规则：

1. 主边界必须包含思明区主体和鼓浪屿；
2. 边界几何需要合法化处理；
3. 多面边界允许存在；
4. 鼓浪屿不与厦门岛主体强行连接；
5. 边界用于裁剪道路和生成分析网格；
6. 边界来源、获取时间和许可说明写入数据版本记录。

## 5. 路网处理

道路处理步骤：

1. 保留 OSM 原始道路 ID 和 `highway` 类型；
2. 统一道路等级到系统 `road_class`；
3. 根据 `highway` 与访问属性生成 `is_motor_accessible`；
4. 根据 `highway` 与访问属性生成 `is_walk_accessible`；
5. 标记 `is_service`、`is_steps`、`is_private`；
6. 在 `EPSG:32650` 下计算 `length_m`；
7. 将正式几何转回 `EPSG:4326`；
8. 写入候选数据版本。

建议道路表增加或保留以下处理字段：

```text
source_osm_id
source_highway
road_class
access_type
is_motor_accessible
is_walk_accessible
is_private
is_steps
is_service
length_m
```

## 6. 节点与网络图

节点处理用于支持交叉口密度、连通性和中心性指标。

处理规则：

1. 分别构建机动车网络和步行网络；
2. 保留节点编号、网络类型、连接度和交叉口标记；
3. 统计连通子图数量；
4. 统计最大连通子图占比；
5. 鼓浪屿作为真实独立子图保留；
6. 不人为连接跨海网络。

## 7. 500 米网格

网格处理规则：

1. 在 `EPSG:32650` 下生成 500 米网格；
2. 网格覆盖完整思明区边界；
3. 与边界相交后生成正式分析网格；
4. 计算 `area_m2`；
5. 转回 `EPSG:4326` 写入 PostGIS；
6. 标记 `is_boundary_grid`；
7. 标记 `is_included_in_stats`；
8. 记录完全无道路网格数量。

## 8. 数据版本发布

Pipeline 先生成候选版本，候选版本通过审计后才能成为当前版本。

发布规则：

1. 每次处理生成唯一 `version_code`；
2. 每次处理记录 `pipeline_runs`；
3. 失败运行不得设为当前版本；
4. 当前启用版本只能有一个；
5. 旧版本不立即删除；
6. 删除旧版本必须人工审计。

示例版本号：

```text
siming-osm-2026-06-v1
```

## 9. 底图配置处理

底图配置不参与指标计算，只服务前端展示。

处理规则：

1. 天地图作为默认底图；
2. OSM 作为备用底图；
3. 离线空白或简化底图作为兜底；
4. 真实天地图凭证不提交 GitHub；
5. `.env.example` 只保留空变量名；
6. 前端通过统一底图适配层读取配置。

## 10. 输出数据

后续 Pipeline 应输出：

| 输出 | 用途 |
|---|---|
| `data_versions` | 数据版本管理 |
| `pipeline_runs` | 数据处理审计 |
| `roads` | 道路展示与分析 |
| `road_nodes` | 路网结构分析 |
| `analysis_grids` | 500 米网格分析单元 |
| `indicator_definitions` | 指标解释和权重 |
| 审计统计 | 数据质量报告 |

## 11. 验收标准

D5 数据处理方案完成后，D6/D7 实现应满足：

1. 可以复现研究范围边界处理；
2. 可以复现 OSM 路网获取或记录获取快照；
3. 可以生成机动车路网和步行路网标记；
4. 可以在 `EPSG:32650` 下完成米制计算；
5. 可以生成 500 米网格；
6. 可以输出数据质量审计统计；
7. 可以写入带版本的数据结构；
8. 可以解释鼓浪屿独立网络现象；
9. 不把底图凭证提交到公开仓库。

## 12. 结论

D5 数据处理方案为后续 Pipeline 提供了可执行路径。正式实现时应坚持“边界可复现、路网可审计、坐标计算正确、底图凭证安全、数据版本可追踪”的原则。
