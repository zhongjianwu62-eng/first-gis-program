# D2 需求分析规格说明

## 厦门市思明区城市路网结构与可步行性分析平台

| 属性 | 内容 |
|---|---|
| 文档版本 | 1.0 |
| 日期 | 2026-06-23 |

## 1. 需求目标

平台应支持专业用户分析思明区机动车路网的城市骨架与关键道路，并基于步行路网评价 500 米网格的路网结构层面可步行性潜力。

用户应能够完成：

1. 观察机动车与步行路网总体结构；
2. 识别边介数中心性较高的关键机动车道路；
3. 查看 500 米网格的五项指标与综合评价；
4. 查询网格得分、排名百分位及相对全区平均值的差异；
5. 识别并定位高值、低值网格；
6. 按道路名称模糊检索并定位路段；
7. 筛选道路或网格并导出当前结果；
8. 在外网中断时继续演示核心分析功能。

## 2. 用户角色与权限边界

首版只设置一种业务角色：**匿名专业用户**。

用户无需登录，可执行：

- 浏览地图与专题图层；
- 切换指标和底图；
- 筛选、搜索、点选和查看详情；
- 查看统计图表与高低值榜单；
- 导出当前筛选结果。

用户不可执行：

- 修改数据库；
- 上传或更新数据；
- 在浏览器中重新计算指标；
- 调整综合评分权重；
- 管理用户或权限。

数据更新、清洗、指标计算和入库由维护人员通过受控脚本执行。

## 3. 系统边界

```text
OSM 固定数据快照
        ↓
人工执行数据处理脚本
        ↓
清洗、建图、指标计算
        ↓
PostGIS 存储分析结果
        ↓
FastAPI 只读查询服务
        ↓
Vue 3 GIS 分析工作台
        ↓
查询、筛选、解释、导出
```

### 3.1 系统输入

- 思明区行政边界；
- OSM 机动车路网；
- OSM 步行路网；
- 固定且公开的指标权重；
- 数据版本与处理过程元数据。

### 3.2 系统输出

- 道路等级专题图；
- 道路边介数中心性专题图；
- 500 米网格单项指标与综合评价图；
- 概览指标卡、指标分布柱状图、网格雷达图；
- 高值与低值网格榜单；
- 道路和网格详情；
- 道路层或网格层当前筛选结果 CSV。

### 3.3 系统外能力

- 用户登录、角色和机构权限；
- 浏览器端数据更新或指标重算；
- OSM 自动定时同步；
- 路径规划和导航；
- 实时交通和人流分析；
- 自动生成规划建议；
- 完整设施可达性评价；
- 任意城市在线建图和分析。

## 5. 核心业务用例

### 5.1 UC-01 分析道路中心性

| 项目 | 内容 |
|---|---|
| 参与者 | 匿名专业用户 |
| 目标 | 识别承担重要网络连接作用的机动车道路 |
| 前置条件 | 数据版本已加载，API 与数据库可用 |
| 触发 | 用户选择边介数中心性分析模式 |

主流程：

1. 系统加载机动车道路中心性图层。
2. 用户查看分级设色和图例。
3. 用户设置中心性等级或数值范围。
4. 地图和统计结果同步更新。
5. 用户点选道路。
6. 系统高亮道路并显示详情。
7. 用户可导出当前筛选结果。

异常分支：

- 接口失败时保留当前地图状态并提供重试；
- 筛选无结果时清除旧结果并显示空状态；
- 无名称道路使用道路编号和“未命名道路”展示；
- 中心性缺失时标记为“暂无计算结果”，不得按零值处理。

关联需求：`FR-NET-002` 至 `FR-NET-009`、`FR-EXP-001`、`FR-SYS-001` 至 `FR-SYS-003`。

### 5.2 UC-02 查询网格可步行性潜力

| 项目 | 内容 |
|---|---|
| 参与者 | 匿名专业用户 |
| 目标 | 理解指定网格的综合得分及其构成 |
| 前置条件 | 五项指标和综合评价已计算入库 |
| 触发 | 用户点选分析网格 |

主流程：

1. 系统高亮网格；
2. 显示综合得分和等级；
3. 显示五项原始指标和单位；
4. 显示五项标准化指标；
5. 显示全区排名百分位；
6. 显示相对全区平均值的差异；
7. 显示网格雷达图；
8. 用户可查看指标定义和评价限制。

异常分支：

- 边缘网格显示有效相交面积或边缘标识；
- 指标缺失时不生成误导性综合分；
- 请求失败时保留选中位置并提供重试；
- 快速连续点选时只采用最后一次有效响应。

关联需求：`FR-WALK-001` 至 `FR-WALK-010`、`FR-GRID-001` 至 `FR-GRID-008`。

### 5.3 UC-03 识别低值区域

主流程：

1. 系统显示低值网格榜单；
2. 用户点击榜单项目；
3. 地图定位并高亮网格；
4. 系统打开网格详情；
5. 用户查看拉低综合得分的单项指标；
6. 用户切换单项指标图层核对；
7. 用户导出当前低值网格结果。

异常分支：

- 筛选范围过窄时提示调整条件；
- 网格不在当前范围时自动定位；
- 数据版本无有效排名时隐藏榜单并解释原因；
- 离线状态下继续使用本地业务数据。

关联需求：`FR-WALK-009` 至 `FR-WALK-011`、`FR-STAT-003` 至 `FR-STAT-005`、`FR-EXP-002`。

### 5.4 UC-04 搜索并查看道路

主流程：

1. 用户输入至少两个字符；
2. 系统返回模糊匹配结果；
3. 用户选择目标道路；
4. 地图缩放到道路范围；
5. 系统高亮同名道路的相关路段；
6. 详情面板显示道路属性和中心性。

异常分支：

- 无匹配结果时不移动地图；
- 同名道路列出候选结果；
- 输入过短时不立即查询；
- 查询失败时保留输入并允许重试。

关联需求：`FR-NET-007` 至 `FR-NET-009`、`FR-SYS-002`、`FR-SYS-003`。

### 5.5 UC-05 离线演示核心功能

主流程：

1. 系统检测在线底图失败；
2. 切换本地简化底图或空白底图；
3. 行政边界、路网、网格和统计继续显示；
4. 用户继续执行指标切换、筛选和查询；
5. 系统提示当前处于离线模式；
6. CSV 导出保持可用。

异常分支：

- 后端或数据库不可用时不展示过期残留结果；
- 本地底图失败时使用纯色背景并保留业务图层；
- 网络恢复后允许手动切回在线底图。

关联需求：`FR-MAP-004`、`FR-DATA-004`、`FR-SYS-003` 至 `FR-SYS-005`。

## 6. 数据需求

### 6.1 数据清单

| 编号 | 数据 | 用途 | 首选来源 |
|---|---|---|---|
| DS-001 | 思明区行政边界 | 裁剪、网格生成、地图范围 | 权威行政区数据，OSM 备用 |
| DS-002 | 机动车路网 | 道路等级、城市骨架、边介数中心性 | OpenStreetMap |
| DS-003 | 步行路网 | 路口、路段、连通性和覆盖计算 | OpenStreetMap |
| DS-004 | 500 米分析网格 | 汇总单项指标和综合评价 | 项目生成 |
| DS-005 | 数据版本元数据 | 复现、审计、界面和导出标识 | 数据流水线生成 |

### 6.2 双路网口径

机动车路网用于城市骨架和关键道路分析，原则上包括主要机动车道路、普通城市道路及合理的连接道路。具体纳入规则在 D5 数据审计阶段确认。

步行路网用于可步行性潜力评价，包括普通道路中允许步行的路段，以及 `footway`、`pedestrian`、`path`、`steps` 等明确支持步行的道路。明确禁止步行或仅限机动车通行的路段不得进入步行网络。

两类路网必须保存独立的 `network_type` 字段，并在数据表、接口和前端图层中分离。

### 6.3 核心字段

道路至少包含：

- `edge_id`、`osm_id`、`network_type`；
- `name`、`highway`、`length_m`；
- `oneway`、`access`、`foot`；
- `bridge`、`tunnel`、`layer`；
- `betweenness`、`betweenness_norm`；
- `geometry`。

节点至少包含：

- 节点唯一标识；
- 网络类型；
- 节点度数；
- 有效路口标识；
- 点几何。

网格至少包含：

- 网格唯一标识和有效面积；
- 五项原始指标；
- 五项标准化指标；
- 综合得分和评价等级；
- 全区排名和百分位；
- 数据版本和网格几何。

### 6.4 坐标系

- 原始数据和接口输出使用 `EPSG:4326`；
- PostGIS 中保存明确的空间参考标识；
- 长度、面积和 500 米网格计算使用米制投影，候选为 `EPSG:32650`；
- D5 通过范围与误差检查确认正式分析投影；
- 禁止直接用经纬度计算长度和面积。

### 6.5 数据版本

每次正式处理生成唯一版本，例如：

```text
siming-osm-20260623-v1
```

版本记录包括：

- 来源、下载时间和查询参数；
- 工具和主要依赖版本；
- 坐标系；
- 原始文件校验值；
- 处理脚本版本；
- 处理时间；
- 道路、节点和网格数量。

界面、API 和 CSV 均应引用当前数据版本。

### 6.6 本地化流程

```text
获取行政边界
    ↓
下载机动车与步行路网
    ↓
保存不可修改的原始数据
    ↓
字段清洗与坐标转换
    ↓
拓扑和几何质量检查
    ↓
分别构建机动车图与步行图
    ↓
计算路段、节点与中心性指标
    ↓
生成 500 米网格并汇总指标
    ↓
标准化与综合评分
    ↓
写入 PostGIS
    ↓
生成数据字典、质量报告和版本清单
```

文件分为 `raw`、`interim`、`processed` 和 `metadata`。完整大数据不进入普通 Git 历史，仓库至少保存获取与处理脚本、数据说明及校验信息。

### 6.7 数据质量检查

- 行政边界完整覆盖鼓浪屿；
- 无空值、无效和重复几何；
- 异常短路段、长路段可解释；
- 两类网络的连通分量合理；
- 厦门岛和鼓浪屿之间不存在虚假道路连接；
- 桥梁、隧道和不同 `layer` 不被错误相交；
- 记录道路名称、类型和权限字段缺失比例；
- 指标不存在未处理的空值、无穷值或异常分布；
- 网格边缘有效面积可识别；
- 处理前后要素数量变化有审计记录。

| NFR-PERF-003 | 常用查询 | 95% 道路搜索和网格详情请求不超过 1 秒 |

### 7.5 安全

- 密钥和密码不得进入代码或 Git 历史；
- 使用环境变量和 `.env.example`；
- 数据库使用最小权限账户；
- 前端不得直连 PostGIS；
- API 输入进行类型、范围和长度校验；
- 道路搜索防止 SQL 注入；
- CSV 导出处理电子表格公式注入风险；
- 生产环境限制跨域来源；
- 错误响应不泄露 SQL、数据库地址或堆栈；
- 提交前执行敏感信息扫描。

首版只提供公开只读数据，不处理个人隐私数据。

### 7.6 可维护性与可测试性

- 前端、API、数据处理和数据库迁移相互独立；
- API 使用 `/api/v1` 版本前缀；
- 核心算法不嵌入页面组件；
- 指标定义、单位、方向和权重集中配置；
- 数据库结构变更使用版本化迁移；
- 固定主要依赖版本；
- README 可指导新开发者启动系统；
- 测试覆盖指标计算、拓扑检查、API、搜索、CSV、核心界面流程、离线兜底和数据版本一致性；
- 核心算法使用固定小样本和预期结果进行回归测试。

### 7.7 可观测与审计

- API 请求具有请求标识；
- 日志记录接口、状态码、耗时和数据版本；
- 日志不得记录密钥或密码；
- 数据任务记录输入版本、脚本版本、输出数量和异常；
- 每天使用独立分支；
- 重要变更小步提交；
- 合并前执行变更、格式、测试和敏感信息审计；
- 每日工作日志记录目标、决策、产物、测试和遗留问题；
- GitHub 保留分支、提交和合并历史。

### 7.8 商业化演进约束

- FastAPI 保持无状态；
- 查询接口可增加缓存而不改变前端业务；
- 地图数据可从 GeoJSON 升级为 MVT；
- 数据库建立必要空间索引；
- 耗时计算不放入普通查询请求；
- 后续可增加限流、多实例、负载均衡和监控。

## 8. 可行性分析

### 8.1 技术可行性

Vue 3、MapLibre GL JS、FastAPI、PostGIS、OSMnx、GeoPandas 和 NetworkX 能覆盖首版需求。主要风险是路网拓扑、边介数中心性计算成本和地图数据量，可通过数据审计、离线预计算、空间索引和按范围加载控制。

### 8.2 数据可行性

OSM 可提供思明区机动车和步行路网，但必须重点核验鼓浪屿步行道路、桥隧和层级、步行权限、道路名称缺失、岛屿间异常连接和行政区边缘截断。成果应明确基于固定 OSM 快照。

### 8.3 工期可行性

20 天、每天 4–6 小时，总投入约 80–120 小时。D1–D4 冻结需求设计，D5–D8 打通最小闭环，D9–D14 完成核心功能，D15 后停止随意增加功能。

### 8.4 运行与部署可行性

同时准备公网版本、本地完整版本、离线业务数据、兜底底图及数据库备份恢复说明。公网或在线底图故障不应阻断核心演示。

## 9. 需求优先级

- **Must**：地图与双底图、双路网、中心性、500 米网格、五项指标、查询、解释、筛选、统计图表、高低值榜单、CSV、离线能力；
- **Should**：在不影响 Must 验收的前提下进行额外视觉与交互打磨；
- **Could**：区域比较、更多中心性、权重敏感性、GeoJSON 导出；
- **Won't**：登录权限、导航、实时交通、自动规划建议、任意城市在线分析。
