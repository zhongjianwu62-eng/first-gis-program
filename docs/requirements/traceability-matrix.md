## 维护规则

1. D3 和 D4 完成后补充具体设计章节或文件。
2. 开发开始后补充实现目录、主要模块和提交编号。
3. 测试完成后补充测试文件、结果和证据。
4. 任意首版功能如果没有需求编号，必须先走需求变更流程。
5. 需求状态只能使用：需求已确认、设计完成、实现完成、测试通过、验收通过、已延期。
